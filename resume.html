Introdução ao desenvolvimento móvel

Sistema Operacional Android
● Baseado em Linux - Android é Linux, apesar de ser majoritariamente usado em celulares e tablets e não em computadores, como ocorre com os sistemas Linux que a maioria das pessoas conhece.

● Para dispositivos móveis - O sistema android se tornou o sistema mais popular do mundo por conta da sua maior compatibilidade.

● Atualmente na versão 13 - Android 13 é o mais recente em 2022. Ele foi lançado na segunda-feira, 15 de agosto de 2022, pelo Google.

● Emulador oficial - O Android Studio é o emulador oficial para desenvolvimento android. É caracterizado por ser muito útil em testar aplicativos em vários dispositivos e configurações de hardware. Porém, sua desvantagem é a complexidade de configuração e seus requisitos para funcionar em alguns computadores.

Arquitetura Android
● Kernel Linux - O sistema android é baseado no kernel linux. permite que o Android aproveite os recursos de segurança principais e que os fabricantes dos dispositivos desenvolvam drivers de hardware para um kernel conhecido.

● Camada de abstração de hardware (HAL) - É uma divisão lógica do código que serve como uma camada de abstração entre o hardware físico de um computador e seu software. Ele fornece uma interface de driver de dispositivo, permitindo que um programa se comunique com o hardware.

● Android Runtime - É um sistema de ambiente de execução usado no Android 5.0 e em versões posteriores, substituindo a máquina virtual Dalvik. O ART, por sua vez, foi concebido para usar uma técnica de compilação chamada AOT.

● Bibliotecas C/C++ nativas - Se estiver desenvolvendo um aplicativo que exige código C ou C++, você pode usar o Android NDK para acessar algumas dessas bibliotecas de plataforma nativa diretamente do seu código nativo.

Google Play
● Loja de aplicativos - Na hora de fazer a build do sistema desenvolvido, é gerado um apk, que através dele, podemos fazer a publicação na google play. No IOS é totalmente diferente, quando não dá para gerar um arquivo apk, você envia para a loja e faz a instalação.

● Atualmente possui taxa única para cadastro - Todo desenvolvedor que for lançar um aplicativo na google play, ele paga uma taxa única. Depois de pagar essa taxa única, ele pode lançar quantos aplicativos quiserem na google play.

Android SDK
● KIT de Desenvolvimento de Software - O Android SDK é um kit de desenvolvimento de software que inclui um conjunto abrangente de ferramentas de desenvolvimento. No React Native, para utilizar a versão web, não é necessário instalat o SDK, pois, as dependências para rodar na versão web, serão instaladas pelo Node Js, via npm. Entretando, se utilizar o Android Studio, é necessário isntalat o SDK para poder desenvolver utilziando o emulador.

Obs: No desenvolvimento em Ios, precisa do Kit Xcode para desenvolver em Ios, transformando a aplicação gerando um arquivo, para poder enviar para loja e fazer publicação.

Integração de Serviços
● Exemplo
– Consumindo API Rest
– Comunicação com WEB Socket

Vamo supor que vamos criar uma aplicação de um Chat, para fazer um chat vamos bater em um endpoint (Um endpoint de API é uma das extremidades de uma conexão de API, onde são recebidas as chamadas de API.) para verificar usuário e senha no backend em uma api, ou seja, estamos consumindo uma API Rest.

Na parte de chat, eu quero que ele seja em tempo real, sem precisar perguntar o servidor se chegou mensagem toda hora, então quando trabalhamos com WebSocket, ele faz a resposta automaticamente, sem ficar perguntando(Sem fazer um Request) toda hora

Quando trabalhamos com APi, é sempre um Request e response.

Pra quem não sabe o que é Request e Response, é um método de comunicação entre computadores em rede. É utilizado na internet e você, desenvolvedor de aplicações de voz, precisa saber como funciona, pelo menos conceitualmente. Afinal, uma skill é construída sobre uma arquitetura web.
Responsividade
● Ajuste da tela para diversos tamanhos de telas

Um exempo é esse site com html e css que eu to fazendo. Ele está responsivo, ou seja, se eu to acessando no meu computador o meu site, ele está adequado a minha tela, sem ficar com partes bugadas da tela. E se eu quiser acessar no Android ou outro sistema que tenha uma tela mutio grande ou até muito pequena, o meu site consegue se adequar ao tamanho de tela, fazendo com que apareça a tela toda, ou seja, ele fica responsivo a todos os tamanhos de tela.

Mesma coisa acontece usando uma aplicação de React Native, o aplicativo ele fica responsivo em todos os sistemas como IOS e Android.

Node JS
● Ambiente de execução de código JavaScript, usando para criar e executar a aplicação mobile em desenvolvimento.

Vamos do começo. Vamos apenas falar de Java Script. O Java Script ele é client-side correto? Então, podemos usar no navegador, por padrão ele é do navegador.

O Node JS vamos fazer com que o Java Script que só é executado no navegador, passa a ser executado sem precisar do navegador, ou seja, ele é executado no terminal.

Para usar o React Native, é obrigatório ter o Node Js instalado, pois vamos desenvolver com java Script fora do navegador.

NPM
● Gerenciador de pacotes utilizado no NodeJS - Já vem instalado junto com o Node Js. Necessário para fazer importações de bibliotecas, baixando as mesmas.

Persistência de dados - Componentes

Componentes
Elemento a ser utilizado na aplicação, podendo ter uma parte visual ou não, que pode ser reutilizada e ser um elemento em uma página, ou em outra página.
Ciclo de vida
Todo processo de atualização/execução de um componente.
Renderização
Mostrar e atualizar conteúdo em tela.
Virtual DOM
Árvore de componentes para renderização como a DOM do Html.
Estado
Valor a ser usado na renderização e armazenamento em memória.
Componentes Stateless
Componente estático.
Componentes Statefull
Componente com estado dinâmico, completo.
Props
São as propriedades, definindo o componente e propriedades para ele.
Child
São os componentes filhos.

useState
Estado de um componente.
Exemplo:

const [title, settile] = useState('')
